# Role-playing system based on RAG_new（项目结构详解）

1. ## 📂 项目结构详解（按功能分类）

   ### 1. 后端核心相关

   - **`app/`**：项目的核心后端代码目录，包含 FastAPI 路由、业务逻辑、RAG 核心实现、数据库模型等，是整个系统的 “大脑”。

             ## 	📂 `app/` 目录详解（后端核心）
        
             ### 	1. 主文件
        
             - ​	**`main.py`**：后端服务的**入口启动文件**。运行它会启动 FastAPI 服务，加载所有路由和配置，是整个系统的启动开关。
               - **`__init__.py`**：Python 包的标识文件，让 `app/` 目录被识别为一个 Python 模块，方便内部文件互相导入。
        
             ------
        
             ### 	2. 按功能分层的核心目录
        
             |     文件夹      |      作用说明      |                           关键职责                           |
             | :-------------: | :----------------: | :----------------------------------------------------------: |
             |     `api/`      |     接口路由层     | 定义所有对外的 API 接口（如对话接口、角色管理接口、知识库上传接口），处理 HTTP 请求和响应，是前后端交互的入口。 |
             |     `core/`     | 核心配置与公共组件 | 存放系统核心配置（如数据库连接、OpenAI 配置、安全设置）和公共工具（如日志、异常处理），是整个后端的 “基础支撑层”。 |
             |      `db/`      |     数据库相关     | 存放数据库模型（如角色表、对话记录表、用户表）和数据库连接配置，负责和数据库的交互。 |
             | `repositories/` |     数据访问层     | 封装数据库操作逻辑（CRUD），和 `db/` 配合，把数据库操作和业务逻辑分离，让代码更易维护。 |
             |   `schemas/`    | 数据模型 / 校验层  | 定义请求和响应的数据格式（用 Pydantic 实现），对前端传入的数据做校验（如角色信息、对话请求），保证数据的合法性。 |
             |   `services/`   |     业务逻辑层     | 存放核心业务逻辑，比如 RAG 检索逻辑、角色对话生成逻辑、知识库处理逻辑，是系统的 “大脑”。 |
        
             ------
        
             ### 	3. 缓存目录
        
             - **`	__pycache__/`**：Python 运行时自动生成的缓存文件，用来加速代码运行，和项目逻辑无关，可以忽略或删除。
        
               ## 💡 关键使用建议
        
               - **调试接口**：修改 `api/` 目录下的文件，添加 / 修改接口路由；
        
               - **核心业务逻辑**：修改 `services/` 目录下的文件，比如调整 RAG 检索策略、优化角色对话回复；
        
               - **数据校验**：修改 `schemas/` 目录下的文件，调整请求 / 响应的数据格式和校验规则；
        
               - ##### **数据库操作**：修改 `repositories/` 和 `db/` 目录下的文件，调整数据库表结构或操作逻辑。

   - **`main.py`**：项目的入口启动文件，运行它即可启动 FastAPI 服务，启动后你的网页就能正常访问了。

   - **`tests/`**：单元测试目录，用来写代码测试，验证各个模块（比如检索、对话接口）的功能是否正常。

   ### 2. 前端与静态资源相关

   - **`frontend/`**：前端页面代码目录，就是你浏览器里看到的那个角色对话界面，包含 HTML、CSS、JavaScript 文件。
   - **`uploads/`**：用户上传文件的存储目录，比如你在系统里上传的知识库 PDF、TXT 文件，都会被存在这里。
   - **`data/`**：项目的基础数据目录，一般用来存放示例知识库文件、语料数据等。

   ### 3. 工具与测试相关

   - **`jmeter/`**：性能测试工具（JMeter）的脚本目录，用来给你的系统做压测，模拟高并发请求，测 QPS 和稳定性。
   - **`postman/`**：接口测试工具（Postman）的脚本目录，用来测试你的 API 接口，比如对话接口、知识库上传接口等。
   - **`scripts/`**：通用脚本目录，存放一些辅助脚本，比如批量处理数据、初始化数据库、清理缓存等。
   - **`log/`**：日志文件目录，系统运行过程中的日志（比如错误信息、请求记录）都会被存在这里，方便排查问题。

   ### 4. 环境与配置相关

   - **`.venv/`**：Python 虚拟环境目录，你的所有项目依赖（比如 langchain、ragas、fastapi）都安装在这里，和系统 Python 环境隔离。
   - **`.env` / `.env.example`**：环境变量配置文件，用来存密钥、数据库地址等敏感信息（比如你的 OpenAI API Key、数据库密码），`.env.example`是示例模板。
   - **`.gitignore`**：Git 配置文件，用来指定哪些文件 / 文件夹不需要被 Git 管理（比如`.venv`、`log`、临时文件）。

   ### 5. 文档与辅助文件

   - **`docs/`**：项目文档目录，存放你的项目说明、API 文档、架构设计等，适合写毕设或项目报告用。
   - **`requirements.txt`**：Python 依赖清单，记录了项目需要安装的所有库和版本，别人拿到你的项目，用`pip install -r requirements.txt`就能一键安装所有依赖。
   - **`rag_eval_result.csv`**：你之前运行 RAG 评测脚本生成的结果文件，记录了评测的各项指标分数。
   - **`项目问答总结.md` / `CHANGELOG.md`**：项目文档文件，`项目问答总结.md`一般是常见问题整理，`CHANGELOG.md`是版本更新记录。

   ### 6. 开发与缓存目录（可忽略）

   - **`.idea/`**：JetBrains IDE（比如 PyCharm）的配置文件目录，和项目本身无关，是 IDE 自动生成的。
   - **`.pytest_cache/` / `__pycache__/`**：Python 运行时生成的缓存文件，用来加速代码运行，可忽略或删除。

   ------

   ## 💡 关键路径提示

   - 启动项目：运行 `main.py`
   - 写接口 / 业务逻辑：修改 `app/` 目录下的文件
   - 改前端界面：修改 `frontend/` 目录下的文件
   - 处理上传的知识库文件：查看 `uploads/` 目录
   - 排查系统问题：查看 `log/` 目录下的日志文件









## 一、后端（FastAPI）核心结构详解

### 1. `app/api/v1/` 接口路由层

这是前后端交互的入口，每个文件对应一组接口：

表格







|      文件       |                     作用说明                     |
| :-------------: | :----------------------------------------------: |
|   `admin.py`    | 管理员后台接口（如角色管理、用户管理、系统配置） |
|    `auth.py`    |     认证接口（登录、注册、Token 生成与校验）     |
| `characters.py` |    角色管理接口（角色创建、修改、删除、查询）    |
|    `chat.py`    |   对话接口（核心，处理用户消息并返回 AI 回复）   |
|   `health.py`   |     健康检查接口（用于监控服务是否正常运行）     |
| `knowledge.py`  |   知识库接口（文件上传、知识库管理、文档处理）   |
|   `router.py`   |        路由聚合文件，统一注册所有接口路由        |

------

### 2. `app/core/` 核心配置与公共组件

系统的基础支撑层：

表格







|        文件        |                        作用说明                         |
| :----------------: | :-----------------------------------------------------: |
|    `admins.py`     |           管理员账号配置（如默认管理员信息）            |
| `blocked_words.py` |             敏感词过滤配置（对话内容风控）              |
|    `config.py`     | 全局配置文件（读取 `.env` 变量、API 密钥、数据库地址）  |
|     `deps.py`      | 依赖注入工具（如 Token 校验、权限控制、数据库会话获取） |
|   `security.py`    |      安全工具（密码加密、JWT 生成、权限校验逻辑）       |

------

### 3. `app/db/` 数据库层

负责数据库连接、模型定义与初始化：

表格







|     文件     |                        作用说明                        |
| :----------: | :----------------------------------------------------: |
|  `base.py`   |   数据库基类（所有 ORM 模型的父类，统一管理表结构）    |
| `init_db.py` |   数据库初始化脚本（创建表、插入初始数据如默认角色）   |
| `models.py`  | 数据库模型定义（用户表、角色表、对话记录表、知识库表） |
| `session.py` |       数据库会话管理（提供数据库连接、事务控制）       |

------

### 4. `app/repositories/` 数据访问层

封装数据库操作，和业务逻辑解耦：

表格







|             文件             |                     作用说明                     |
| :--------------------------: | :----------------------------------------------: |
|  `character_repository.py`   |      角色相关数据库操作（增删改查角色信息）      |
| `conversation_repository.py` | 对话相关数据库操作（保存对话记录、查询历史对话） |
|  `knowledge_repository.py`   |      知识库相关数据库操作（文档元数据管理）      |
|     `user_repository.py`     |   用户相关数据库操作（用户信息管理、登录记录）   |

------

### 5. `app/schemas/` 数据校验层

用 Pydantic 定义请求 / 响应的数据格式：

表格







|      文件      |               作用说明                |
| :------------: | :-----------------------------------: |
|   `auth.py`    |    登录 / 注册请求的数据格式与校验    |
| `character.py` |  角色创建 / 修改请求的数据格式与校验  |
|   `chat.py`    |    对话请求 / 响应的数据格式与校验    |
| `knowledge.py` | 知识库上传 / 查询请求的数据格式与校验 |

------

### 6. `app/services/` 业务逻辑层（核心）

系统的 “大脑”，实现所有核心业务逻辑：

表格







|          文件           |                         作用说明                          |
| :---------------------: | :-------------------------------------------------------: |
|    `chat_service.py`    | 对话服务（处理用户请求、调用 RAG 与 LLM、拼接角色提示词） |
| `knowledge_service.py`  |        知识库服务（文档解析、向量化、向量库存储）         |
|    `llm_service.py`     | 大模型服务（调用 OpenAI / 本地模型、设置温度 / 模型参数） |
|   `memory_service.py`   |       对话记忆服务（维护上下文、处理历史对话拼接）        |
| `pdf_ingest_service.py` |       PDF 处理服务（解析 PDF、提取文本、分块处理）        |
|    `rag_service.py`     |       RAG 检索服务（向量检索、上下文召回、重排序）        |

## 二、前端（React/Vite）结构详解

`frontend/` 目录下是你的角色对话界面代码：

## 💡 关键修改建议（毕设 / 优化用）

1. **优化对话逻辑**：修改 `services/chat_service.py`，调整角色提示词拼接、上下文管理策略；
2. **提升检索效果**：修改 `services/rag_service.py`，调整向量检索 Top-K、重排序策略；
3. **修改前端界面**：在 `frontend/src/` 中调整对话界面样式、角色展示方式；
4. **添加新角色**：在 `db/models.py` 或通过后台接口添加新角色，并在 `data/` 中放入对应知识库文件。
