# 基于 RAG 的角色扮演对话系统 — 需求规格说明书

## 1. 引言

### 1.1 目标

构建可扮演多种角色（医生、律师、虚拟朋友等）的聊天机器人，支持多用户、多角色、多轮对话；具备知识库动态更新与 RAG 检索能力；提供 HTTP JSON 接口供前端或测试工具调用。

### 1.2 范围

本版本交付单进程 FastAPI 服务、MySQL 持久化、Redis 短期记忆、可切换的 Milvus 检索占位实现、可切换的本地 Mock 或 OpenAI 兼容大模型调用；知识库上传落盘与元数据入库，向量化异步流水线可在后续迭代接入。

## 2. 角色与用户

### 2.1 用户管理

- 注册：账号、密码、昵称；密码使用单向哈希存储。
- 登录：校验账号密码，签发 JWT（默认 HS256）。
- 角色偏好：用户可维护 `preferred_character_ids` 列表（JSON 存 MySQL）。

### 2.2 角色管理

系统预置角色类型示例：医疗（高血压医生）、法律（民事律师）、社交 NPC（虚拟朋友）。角色字段包含人设、领域、提示词模板、知识库分区标识。

## 3. 对话与记忆

### 3.1 多轮对话

- 对话按 `(user_id, character_id)` 维度聚合为会话，消息写入 `chat_message`。
- Redis 维护最近 `RAG_SHORT_MEMORY_ROUNDS` 轮（默认 20 轮，每轮用户+助手各一条，List 裁剪）。

### 3.2 会话键规范

短期记忆 Redis Key：`chat:session:{user_id}:{character_id}`。

### 3.3 并发业务规则

每个用户最多同时与 3 个角色保持“活跃对话”。活跃判定：Redis Hash `chat:active_roles:{user_id}` 记录各 `character_id` 最近发言时间戳；超过 `RAG_ACTIVE_ROLE_IDLE_SECONDS`（默认 3600 秒）未发言的角色不占名额。

## 4. 知识库

- 支持按角色上传 `txt` / `pdf` / `md` 文件。
- 元数据写入 `knowledge_document`（状态 `pending`，后续异步向量化写入 Milvus）。
- 删除与版本控制可在后续迭代扩展（当前提供列表查询）。

## 5. RAG 流程（目标架构）

1. 文档切分（段落/固定长度）。
2. BGE-m3 向量化（稠密 + 稀疏，由独立推理服务完成）。
3. Milvus 混合检索（稠密 + 稀疏 / BM25 变体，视 Collection 设计而定）。
4. BGE-rerank 重排序。
5. 与 Redis 短期记忆、角色提示词模板拼接上下文。
6. 调用大模型生成回复。

当前代码在 `RAG_MILVUS_ENABLED=false` 时使用内置 mock 片段，便于无向量环境联调。

## 6. 大模型

- `RAG_LLM_PROVIDER=mock`：模板化占位回复。
- `openai` / `vllm` / `sglang`：走 OpenAI 兼容 `POST {base}/chat/completions`。

## 7. 非功能需求

- 接口：RESTful，JSON；OpenAPI 自动生成文档。
- 测试：Postman 集合、JMeter 压测脚本、pytest 单元测试（安全模块）。
- 评测：RAGAS 指标（见 `docs/RAGAS评测说明.md`），需在离线评测集上执行。

## 8. 验收要点（建议）

- 注册/登录成功，聊天返回 200，历史接口可拉取最近消息。
- Redis 中存在对应 `chat:session:*` 列表。
- 上传知识文件后 `knowledge_document` 有新记录且磁盘存在文件。
- 关闭 Milvus 时系统仍可运行；开启 Milvus 且 Collection 字段匹配时可返回检索文本。

```
def _embed(self, text: str) -> list[float]:
    # 用 SHA256 生成确定性向量
    digest = hashlib.sha256(text.encode("utf-8")).digest()
    vector = []
    for i in range(settings.milvus_dim):  # 默认 128 维
        byte = digest[i % len(digest)]
        vector.append((byte / 255.0) * 2 - 1)  # 映射到 [-1, 1]
    return vector
```

> ⚠️ **注意**：当前用的是 SHA256 哈希伪向量，**不是语义嵌入**。如果要语义检索，需要接入真实 Embedding 模型（如 OpenAI `text-embedding-3-small` 或本地模型）。
