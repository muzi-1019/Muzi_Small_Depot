Transformer 知识点整理
==================================================

一、Input Embedding：文本转向量
--------------------------------------------------

1. 文本 -> Token ID（Tokenizer / 分词器）

【定义】
把原始文本切分成一个个 token，再将每个 token 映射成唯一的整数 ID。

【核心流程】
文本
-> 分词
-> token
-> token ID

【重点】
- Tokenizer 负责“切分 + 编码到 ID”。
- token ID 本质上就是词表中的索引。
- BPE 主要负责学习“怎么切分”，不是单独负责“ID 映射”的那一步。

【常见分词算法】
1. BPE（Byte Pair Encoding）
- 最小单位-字符。
- 通过合并高频相邻符号，形成 token。

2. BBPE（Byte-level BPE）
2. **BBPE 先把一个中文字符，按照 UTF-8 的规则，拆成 3 个独立的“字节”，然后再看这 3 个字节是否能经常一起出现，能的话就合并成一个新的词。**
- 最小单位是字节（Byte）。
- 例如中文字符通常会先按 UTF-8 拆成3字节，再进行合并学习。

【为什么中文常用 BBPE】
- BPE 遇到生僻字时，可能会出现无法稳定覆盖的问题。
- BBPE 从字节出发，理论上可以覆盖任意文本。
- 即使遇到未见过的字，也能退回到字节级表示。

【基本原理】
- 先从最小单位开始拆分。
- 统计相邻符号对的出现频率。
- 每轮合并频率最高的符号对。
- 合并后的新单元进入词表，并分配对应 ID。
- #### 3. 所以“拆成 3 字节”是什么意思？

  - 在 BBPE 刚开始处理文本时，它看不到“中”这个字符。
  - 它看到的是三个独立的字节：`[228]` `[184]` `[173]`。
  - 这就是“拆成 3 字节”的意思。

  #### 4. 之后怎么合回去？

  BBPE 会统计所有字节序列，发现 `[228, 184]` 经常一起出现，就会把它们合并成一个新词（比如叫 `BytePair_1`）。
  接着发现 `BytePair_1` 和 `[173]` 也经常一起出现，就再合并成 `BytePair_2`。
  最终 `BytePair_2` 就等于原来的“中”字了。

【训练阶段】
- 不断学习“哪些相邻符号应该合并”。
- 逐轮扩充或确定词表。

【推理阶段】
- 按训练时学到的合并规则，依次尝试合并输入文本。
- 最终把得到的 token 映射到词表中的 ID。

【易错点】
- BPE 不是“直接生成向量”的算法，它只处理分词与子词构造。
- token ID 不是语义本身，只是后续查表时使用的编号。


2. token ID -> 向量（Embedding Layer / 嵌入层）

【定义】
嵌入矩阵本质上是一个巨大的查找表（Lookup Table）。

【查表方式】
- 行号：token ID
- 行内容：该 token 对应的稠密向量

【重点】
- 输入神经网络的不是 token ID 本身，而是 token ID 对应的向量。
- 一个 token 对应一行 embedding。
- embedding 向量中的数值是训练得到的，不是人工写死的。


二、Positional Encoding：位置编码
--------------------------------------------------

【定义】
位置编码用于给输入向量补充“顺序信息”，让模型知道谁在前、谁在后。

【为什么必须加位置编码】
- Transformer 的核心注意力计算本身不天然理解顺序。
- 注意力更关注“内容相关性”，而不是“位置先后关系”。
- 如果不加位置信息，模型很难区分：
  “我爱你” 和 “你爱我”。

【加入方式】
位置编码向量与词向量逐元素相加：

输入向量 = Token Embedding + Positional Encoding

【重点】
- 位置编码向量的维度必须和 embedding 维度一致。
- 位置编码不是替代词向量，而是补充位置信息。
- 真正对位置不敏感的核心原因，不是“多头”本身，而是点积注意力本身不包含顺序偏置。

【易错点】
- “多头注意力识别不了位置”这种说法不够准确。
- 更准确地说：Scaled Dot-Product Attention 本身不带位置信息，所以需要额外注入位置编码。


三、Multi-Head Attention：多头注意力
--------------------------------------------------

【定义】
把输入映射到多组不同的 Q、K、V，在不同子空间里分别做注意力计算，再把结果拼接（横向）起来。

【标准流程】
输入 X
-> 分别乘以不同权重矩阵，得到多组 Q、K、V
-> 每一组独立计算 attention，得到一个 Z
->横向 拼接所有注意力 （Z1~Z8)
-> 再乘输出矩阵 WO
-> 得到最终结果 Z

->与softmax 相乘

【重点】
- 多头机制能让模型从不同角度捕捉语义关系。
- 不同 head 可能关注不同信息：局部关系、远距离依赖、语法模式等。
- 单头看问题比较单一，多头更像“多个观察角度同时判断”。

【优点】
- 增强表达能力
- 提升稳定性
- 提高对复杂模式的建模能力

【关于 sqrt(dk)】
- 注意力分数通常会除以 sqrt(dk)。
- 目的不是“对 K 开方”，而是对点积结果做缩放。
- 这样可以避免数值过大，防止 softmax 过早饱和。


四、Add & Norm：残差连接与层归一化
--------------------------------------------------

1. Add（残差连接）

【公式】
Add = x + z

其中：
- x：原始输入
- z：多头注意力输出

【作用】
- 保留一部分原始信息
- 缓解深层网络中的梯度消失问题
- 让训练更稳定


2. Norm（Layer Normalization / 层归一化）

【本质】
对每个样本自身的特征做归一化，使其整体分布更稳定。

【目标】
- 均值约为 0
- 方差约为 1

【重点】
- LayerNorm 是“对单个样本内部”做归一化。
- BatchNorm 是“对一个 batch 的同一特征”做归一化。
- Transformer 里更常用 LayerNorm。

【运算原理】选看
假设输入向量：

A = [1.1, 0.8, -0.3]

第 1 步：计算均值
μ = (1.1 + 0.8 - 0.3) / 3 = 0.533333...

第 2 步：计算方差
σ^2 = [(1.1 - μ)^2 + (0.8 - μ)^2 + (-0.3 - μ)^2] / 3
    = (0.32114 + 0.07111 + 0.69444) / 3
    = 0.36223

第 3 步：计算标准差
σ = sqrt(σ^2 + ε)
  = sqrt(0.36223 + 0.00001)
  = 0.60186

第 4 步：归一化
(x - μ) / σ

得到：
- (1.1 - 0.53333) / 0.60186 = 0.9415
- (0.8 - 0.53333) / 0.60186 = 0.4431
- (-0.3 - 0.53333) / 0.60186 = -1.3846

归一化结果：
[0.9415, 0.4431, -1.3846]

第 5 步：缩放和平移
最终输出 = γ * 归一化结果 + β

其中：
- γ（gamma）：可学习的缩放参数
- β（beta）：可学习的平移参数

【结论】
- 归一化不是简单“改数值”，而是在稳定训练过程。
- γ 和 β 让模型在归一化后仍保留足够的表达自由度。


五、Feed Forward Network：前馈神经网络
--------------------------------------------------

【定义】
在每个位置上独立作用的全连接网络，通常位于注意力层之后。

【常见结构】两层全连接网络 中间加一个激活函数
     Linear             升维 一般扩大到原来四倍 在高维空间学习更多特征
-> 激活函数（如 ReLU / GELU）swgLu

-> Linear             降维  降到与输入的x向量 维度一样 方便后续的向量堆叠

【作用】
- 增强非线性表达能力


【重点】选看
- 注意力层负责“信息交互”。
- FFN 负责“特征变换”。
- FFN 虽然不在 token 间交互，但对表示能力非常关键。


六、Masked Multi-Head Attention：掩码自注意力
--------------------------------------------------

【核心作用】
防止大模型在预测时能看到未来，这样模型就会“作弊”。

【工作方式】
- 通过一个上三角掩码矩阵，把未来位置的注意力分数屏蔽掉。
- 被屏蔽的位置在 softmax 前会变成极小值。
- 这样当前位置最终只能关注自己以及自己之前的位置。

【重点】
- Mask 的核心不是“少算一些”，而是“禁止访问未来信息”。
- 这一步是解码器能够自回归生成文本的关键。

【总结】
Masked Self-Attention 保证了 Transformer 解码器在训练时虽然可以并行计算，但行为逻辑仍然符合逐步生成的要求。


七、Cross Attention：交叉注意力
--------------------------------------------------

【定义】
交叉注意力通常出现在编码器-解码器结构中：

- Q 来自解码器
- K、V 来自编码器输出

【作用】
让解码器在生成目标序列时，能够读取输入序列的信息。

【重点】
- 自注意力：Q、K、V 都来自同一个序列
- 交叉注意力：Q 来自解码器，K / V 来自编码器


八、LM Head：输出层 / 词表映射层
--------------------------------------------------

【定义】
LM Head 通常是**最后的线性层**，用来把隐藏向量映射到整个词表大小的维度上。

【示例维度】
- 词表大小：248044 个 token
- 隐藏维度：1024 维

【矩阵计算】
[1 x 1024] x [1024 x 248044] = [1 x 248044]

含义如下：
- [1 x 1024]：当前时刻的隐藏状态向量 - 由解码器输出的token
- [1024 x 248044]：映射到词表空间的权重矩阵
- [1 x 248044]：对词表中每个 token 的打分结果（logits）

【重点】
- 这里得到的不是概率，而是 logits（原始分数）。
- 经过 softmax 后，才会变成概率分布。
- 概率最大的那个 token，就是当前步的预测结果。

【Softmax 后做什么】
- 找到最大概率对应的索引
- 再去词表中查这个索引对应的 token
- 最终得到可读文本

【补充说明】
- “根据索引查词表”这一步通常属于模型外部的程序逻辑。
- 神经网络本身输出的是分数或索引，不直接输出人类语言字符串。

【易错点】
- LM Head 不负责“理解语义”，它负责把隐藏表示映射回词表空间。
- 最后一层线性层常常与 Embedding 层权重共享或相关联，但具体实现要看模型结构。


九、整体系结构速记
--------------------------------------------------

文本
-> Tokenizer
-> token ID
-> Embedding
-> 加入位置编码
-> 多头注意力 / 掩码注意力 / 交叉注意力
-> Add & Norm
-> FFN
-> LM Head
-> Softmax
-> 输出下一个 token

【必须记住】
1. token ID 不是语义，embedding 向量才是模型真正处理的输入。
2. Transformer 天生不懂顺序，所以必须注入位置信息。
3. 多头注意力解决“从不同角度看信息”，不是单纯重复计算。
4. Mask 的本质是保证因果性，防止看到未来。
5. LM Head 的输出先是 logits，经过 softmax 后才是概率。


十、当前这份笔记最容易混淆的点
--------------------------------------------------

【易混淆 1】
BPE、Tokenizer、token ID 不是同一层概念。

- BPE：分词策略
- Tokenizer：分词并编码
- token ID：编码结果

【易混淆 2】
位置编码不是可有可无。
- 没有位置编码，Transformer 无法天然理解顺序。

【易混淆 3】
sqrt(dk) 不是“对 K 开方”。
- 它是对注意力分数做缩放。

【易混淆 4】
LayerNorm 和 BatchNorm 的归一化维度不同。
- LayerNorm：看单个样本内部
- BatchNorm：看一个 batch 的同一维特征

【易混淆 5】
LM Head 输出的不是最终文字。
- 它先输出词表分数，再由程序映射成 token / 文本。
